# Manual Test Validation — Procedimiento de habilitación post-revisión

Esta guía es para el equipo humano. Describe cómo habilitar un test auto-generado por `test-generator` después de revisión manual.

## Contexto

Cuando el pipeline QA no encuentra sessions existentes para un ticket, `test-generator` crea automáticamente un archivo `.test.ts` en `sessions/{domain}/`. Estos tests tienen:
- `// @validated: false` en el cabecero del archivo
- `"validated": false` en `test-map.json`

test-engine **no ejecuta** tests con `validated: false` en ejecuciones automáticas.

## Proceso de habilitación

1. **Revisar el test generado** — abrir `sessions/{domain}/AutoGenerated_{ticket_key}.test.ts`
2. **Verificar que el test:**
   - Cubre los criterios del ticket
   - No tiene aserciones vacías o incorrectas
   - Los comentarios `// MANUAL:` están justificados
3. **Habilitar el archivo:**
   ```typescript
   // @auto-generated: true
   // @ticket: NAA-XXXX
   // @validated: true   // ← cambiar de false a true
   ```
4. **Actualizar test-map.json** — cambiar la entrada del test:
   ```json
   { "validated": true }
   ```
5. **Crear commit:**
   ```
   git commit -m "feat(sessions): habilitar AutoGenerated_NAA-XXXX tras revisión manual"
   ```
6. En la siguiente ejecución del pipeline, test-engine incluirá el test en el ciclo normal.
