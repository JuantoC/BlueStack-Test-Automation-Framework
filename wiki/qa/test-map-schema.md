# test-map.json — Schema

> Fuente canónica de sessions disponibles por módulo. Leído por test-engine (TE-4) para discovery y escrito por test-generator (TG-5) al registrar sessions auto-generadas.

**Path del archivo:** `.claude/pipelines/test-engine/references/test-map.json`

---

## Estructura raíz

```json
{
  "version": "1.0",
  "last_updated": "YYYY-MM-DD",
  "modules": {
    "<module_key>": { ... }
  }
}
```

| Campo | Tipo | Descripción |
|-------|------|-------------|
| `version` | string | Versión del schema |
| `last_updated` | string (fecha) | Fecha de última modificación |
| `modules` | object | Mapa de módulos. Keys válidos: `post`, `ai-post`, `video`, `images`, `auth`, `cross`, `stress` |

---

## Schema por módulo

```json
{
  "sessions": ["NombreSesion", "OtraSesion"],
  "paths": [
    "sessions/post/NombreSesion.test.ts",
    "sessions/post/OtraSesion.test.ts"
  ],
  "page_objects": ["src/pages/post_page/"],
  "keywords": ["keyword1", "keyword2"],
  "component_jira": "Post",
  "validated": true,
  "auto_generated_sessions": [
    {
      "file": "sessions/post/AutoGenerated_NAA-XXX.test.ts",
      "description": "Descripción del caso auto-generado",
      "validated": false,
      "auto_generated": true,
      "ticket": "NAA-XXX",
      "generated_at": "2026-04-17T00:00:00.000Z"
    }
  ],
  "not_automatable_components": [
    {
      "component": "Nombre del componente no automatizable",
      "reason": "reason_code",
      "status": "manual_only",
      "note": "Explicación técnica",
      "keywords": ["keyword que activa esta restricción"]
    }
  ]
}
```

### Campos del módulo

| Campo | Tipo | Obligatorio | Descripción |
|-------|------|-------------|-------------|
| `sessions` | string[] | ✅ | Nombres de sesiones (sin path ni extensión) |
| `paths` | string[] | ✅ | Paths relativos al root del repo de cada `.test.ts` |
| `page_objects` | string[] | ✅ | Directorios POM relevantes para el módulo |
| `keywords` | string[] | ✅ | Keywords para matching semántico (TE-4 nivel 3) |
| `component_jira` | string \| null | ✅ | Nombre del componente en Jira para matching exacto (TE-4 nivel 4). `null` para módulos cross/stress |
| `validated` | boolean | ✅ | `true` si todas las sessions del módulo tienen `@validated: true` en su cabecera |
| `auto_generated_sessions` | object[] | ❌ | Solo presente si el módulo tiene sessions auto-generadas por test-generator |
| `not_automatable_components` | object[] | ❌ | Solo presente si hay sub-componentes del módulo que son `non_automatable` |

### Campos de `auto_generated_sessions[]`

| Campo | Tipo | Descripción |
|-------|------|-------------|
| `file` | string | Path de la session auto-generada |
| `description` | string | Descripción del caso cubierto |
| `validated` | boolean | `false` hasta que el QA ejecute revisión manual y cambie a `true`. Ver [manual-test-validation.md](manual-test-validation.md) |
| `auto_generated` | boolean | Siempre `true` para sessions creadas por test-generator |
| `ticket` | string | Ticket Jira de origen |
| `generated_at` | string (ISO 8601) | Timestamp de generación |

---

## Quién lee y quién escribe

| Agente | Operación | Paso |
|--------|-----------|------|
| **test-engine** | Lee `test-map.json` para discovery de sessions | TE-4 |
| **test-generator** | Agrega una entrada en `auto_generated_sessions[]` del módulo correspondiente | TG-5 |
| **test-engine** | Lee `auto_generated_sessions[].validated` para decidir si ejecutar session auto-generada | TE-5 |

> test-engine ejecuta sessions con `validated: false` solo si el orchestrator lo instruye explícitamente (modo `run_existing`). Por defecto, las sessions `validated: false` se omiten en `discover_and_run`.

---

## Lógica de discovery (TE-4)

Test-engine matchea módulo con precedencia estricta (orden descendente, no cascada):

1. `classification.module` == key en `modules{}`
2. `classification.domain` == key en `modules{}`
3. algún `test_hint` o `acceptance_criteria` contiene keyword del módulo
4. `component_jira` del ticket matchea `component_jira` del módulo

Ver regla completa en `.claude/agents/test-engine.md` §TE-4.
